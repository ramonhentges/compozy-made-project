# ADR-003: Ports & Adapters inside module

## Status

Accepted

## Date

2026-04-15

## Context

Hexagonal Architecture defines driven ports (for persistence/external services) and driver ports (for use cases). We need to define where these live.

## Decision

**Driven ports** (repository interfaces) are defined in `domain/` layer.
**Driver ports** (use case interfaces) are defined in `application/` layer.
**Adapters** (implementations) live in `infrastructure/`.

```
Module:
├── domain/
│   └── repository/
│       └── IUserRepository.ts    # Driven PORT (interface)
├── application/
│   └── use-cases/
│       └── register-user/
│           └── port.ts          # Driver PORT (interface)
└── infrastructure/
    └── persistence/
        └── user-repository.ts   # ADAPTER (implements IUserRepository)
```

## Consequences

- Domain defines what it needs (port interface)
- Application defines how it's used (use case interface)
- Infrastructure provides how (adapter implementation)
- Clear separation following Hexagonal Architecture

## Rationale

This follows the clean-ddd-hexagonal skill reference where ports are defined alongside the layer that consumes them.
