# ADR-004: Manual Dependency Injection

## Status

Accepted

## Date

2026-04-15

## Context

The architecture needs a way to wire dependencies between layers without introducing framework magic or additional dependencies. This follows ADR-001 (module structure) and ADR-002 (domain isolation).

## Decision

Use plain manual Dependency Injection via function parameter passing in the Composition Root at entry point.

```
src/main/index.ts          # Composition Root - wires all dependencies
src/modules/identity/
  application/use-cases/
    register-user/
      handler.ts          # Receives dependencies as constructor params
```

## Alternatives Considered

### Alternative 1: tsyringe

- **Description**: IoC container with decorators (@injectable, @inject)
- **Pros**: Familiar to NestJS users, cleaner code
- **Cons**: Adds dependency, hides wiring logic, learning curve
- **Why rejected**: Over-engineering for MVP, hides traceability

### Alternative 2: inversify

- **Description**: Powerful IoC container for TS/JS
- **Pros**: Well-established, powerful features
- **Cons**: Complex configuration, additional dependency
- **Why rejected**: Not needed at MVP scale

## Consequences

### Positive

- Zero additional dependencies
- Explicit dependency wiring - easy to trace
- No reflection/metadata overhead
- Simple to understand and debug

### Negative

- More boilerplate in composition root
- Manual parameter passing on every call

### Risks

- Risk: Wiring complexity grows with module count
- Mitigation: Keep composition root organized by module
