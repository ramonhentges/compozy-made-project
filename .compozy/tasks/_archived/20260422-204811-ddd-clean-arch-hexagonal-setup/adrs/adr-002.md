# ADR-002: Domain layer isolation

## Status

Accepted

## Date

2026-04-15

## Context

Clean Architecture's core principle is the dependency rule: dependencies point inward toward business logic.

## Decision

Domain layer (`modules/*/domain/`) will have ZERO imports from any other layer. It is framework-agnostic and database-agnostic.

## Rules

1. **Entities** — Plain TypeScript classes, no decorators, no ORM annotations
2. **Value Objects** — Immutable classes with equality logic
3. **Repository Interfaces** — Define contracts (DRIVEN ports) without implementation
4. **Domain Services** — Pure business logic, no external dependencies

## Consequences

- Entities are pure business models — portable to any context
- Testing domain logic requires no mocks of infrastructure
- Repository implementations must satisfy interface contracts exactly
- Violation is an architectural error

## Enforcement

ESLint import rules + module boundary tests will verify isolation.
