# ADR-003: Shared UUID Generation Utility

## Status

Accepted

## Date

2026-04-17

## Context

The UserId value object needs auto-generation via `UserId.create()` with no parameters. The PRD requires using Node.js built-in `crypto.randomUUID()`. To avoid duplication across ID-type value objects, a shared utility is needed.

## Decision

Create a shared UUID generator utility at `src/shared/utils/uuid_generator.ts`:
- Uses Node.js `crypto.randomUUID()` for UUID v4 generation
- Single source of truth for all ID generation in the codebase
- Used by value objects implementing parameterless `create()`

## Alternatives Considered

### Alternative 1: Each module has its own generator

- **Description**: Create uuid_generator.ts in each module
- **Pros**: Encapsulated per module
- **Cons**: Code duplication, inconsistency
- **Why rejected**: PRD requires consistent ID generation across all modules

### Alternative 2: UUID package

- **Description**: Use npm `uuid` package
- **Pros**: Well-tested, supports multiple versions
- **Cons**: Additional dependency, built-in crypto.randomUUID() is sufficient
- **Why rejected**: Unnecessary dependency for simple v4 UUID generation

## Consequences

### Positive

- Single implementation of UUID generation
- Easy to swap generation strategy in one place
- Built-in Node.js means no additional dependencies
- Consistent across all ID-type value objects

### Negative

- Shared utility adds slight indirection

### Risks

- None identified

## Implementation Notes

1. Create `src/shared/utils/uuid_generator.ts`
2. Export function `generateUuid(): string` wrapping `crypto.randomUUID()`
3. UserId.create() calls `generateUuid()` internally
